클래스 다이어그램 (class diagram)

Controller : 컨트롤러는 중간 제어자 역할을 하며, 사용자의 요청을 처리한 후 지정된 뷰에 모델 객체를 넘겨주는 역할을 담당
Config : 개발 중인 프로그램을 각기 다른 수많은 컴퓨터들에서 실행하고 도와주도록 하는 기능

DTO : 계층 간에 데이터 교환을 위한 객체이며, DB에서 데이터를 얻어서 Service 나 Controller 등으로 보낼 때 사용하는 객체

Entity : 실체, 객체라고 생각할 수 있으며 프로젝트에서는 Entity 안에 제목, 사용자, 관리자, 이메일, 비밀번호, 이미지 번호 등으로 설정해두었다.

Repository : Entity에 의해 생성된 DB에 접근하는 메서드들을 사용하기 위한 인터페이스이며, extends를 통해서 상속받고 나면, 해당 레포지토리의 객체를 이용해서 기본적으로 제공되는 메서드를 사용 가능하다.
